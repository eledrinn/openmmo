## Gap B1: Recursive Fixture Discovery

### Current State
```javascript
// From Validator/index.js:54-70
function validateProject(basePath) {
  const itemsDir = path.join(basePath, 'Fixtures', 'items');
  if (fs.existsSync(itemsDir)) {
    results.items = validateDirectory(itemsDir, 'item');
  }
  
  const rewardsDir = path.join(basePath, 'Fixtures', 'rewards');
  if (fs.existsSync(rewardsDir)) {
    results.rewards = validateDirectory(rewardsDir, 'reward');
  }
}
```

**Issue:** Only validates `Fixtures/items/` and `Fixtures/rewards/` at top level. Does not recurse into subdirectories.

### Evidence
```bash
$ ls -R Fixtures/items/
Fixtures/items/:
boundary_01_max_length.json  invalid/  valid/

Fixtures/items/invalid/:
01_missing_required_name.json  ...

Fixtures/items/valid/:
01_consumable_potion.json  ...
```

**Result:** Files in `valid/` and `invalid/` subdirectories are NOT discovered by `validateProject()`.

### Impact
- **Severity:** Medium
- **Layer:** 2 (Semantic)
- **User Impact:** Users must validate subdirectories manually
- **AJV Migration Impact:** None — this is Layer 2 functionality

### Recommended Fix
```javascript
function discoverFiles(dir, pattern = '*.json') {
  const files = [];
  const entries = fs.readdirSync(dir, { withFileTypes: true });
  
  for (const entry of entries) {
    const fullPath = path.join(dir, entry.name);
    if (entry.isDirectory()) {
      files.push(...discoverFiles(fullPath, pattern));
    } else if (entry.name.endsWith('.json')) {
      files.push(fullPath);
    }
  }
  
  return files;
}
```

## Gap B2: Reference Validation Not Integrated by Default

### Current State
```javascript
// From Validator/src/validator.js:80-115
function validateFile(filePath, schemaName) {
  // 1. Load and parse file
  // 2. Load schema
  // 3. validateValue(data, schema)  ← Layer 1 only
  
  return { valid: errors.length === 0, errors, data };
  // NO call to validateReferences()
}
```

**Issue:** `validateFile()` only performs JSON Schema validation (Layer 1). It does NOT call `validateReferences()` for semantic validation (Layer 2).

### Evidence
```javascript
const { validateFile } = require('./Validator');
const result = validateFile('./Fixtures/items/valid/01_consumable_potion.json', 'item');
// Result: { valid: true, errors: [] }
// Note: Does NOT check if 'reward_heal_small' exists
```

The `on_use_reward` reference is NOT validated for existence or target correctness.

### Impact
- **Severity:** High
- **Layer:** 2 (Semantic)
- **User Impact:** Invalid references pass validation silently
- **AJV Migration Impact:** None — reference validation stays in Layer 2

### Recommended Fix
```javascript
function validateFile(filePath, schemaName, options = {}) {
  // Layer 1: JSON Schema validation
  const schemaResult = validateValue(data, schema, '', data);
  
  // Layer 2: Semantic validation (if Layer 1 passes)
  if (schemaResult.length === 0 && options.checkReferences !== false) {
    const refErrors = validateReferences(data, ...);
    schemaResult.push(...refErrors);
  }
  
  return { valid: errors.length === 0, errors, data };
}
```

## Gap B3: Conditional Validation Edge Cases Untested

### Current State
```javascript
// From Validator/src/validator.js:240-256
if (schema.if && schema.then) {
  const ifSchema = schema.if;
  let ifConditionMet = true;
  
  if (ifSchema.properties) {
    for (const [propName, propCondition] of Object.entries(ifSchema.properties)) {
      if (propCondition.const !== undefined) {
        if (value[propName] !== propCondition.const) {
          ifConditionMet = false;
          break;
        }
      }
    }
  }
  
  if (ifConditionMet) {
    const thenErrors = validateValue(value, schema.then, path, rootData);
    errors.push(...thenErrors);
  }
}
```

**Issue:** Conditional validation (if/then) is implemented but has limited test coverage for edge cases.

### Evidence
**Tested:** Equipment with slot passes, equipment without slot fails.

**Not Tested:**
- Nested conditionals (if/then inside if/then)
- Multiple conditional branches
- Stackable conditional (max_stack_size minimum)
- Conditional with additionalProperties

### Impact
- **Severity:** Medium
- **Layer:** 1 (JSON Schema)
- **User Impact:** Edge cases may behave unexpectedly
- **AJV Migration Impact:** AJV handles conditionals correctly — this gap disappears post-migration

### Recommended Fix
Add tests for:
```javascript
// Test B3.1: Equipment conditional
test('Equipment with slot passes', ...);
test('Equipment without slot fails', ...);

// Test B3.2: Stackable conditional
test('Stackable true with max_stack_size 1 passes', ...);
test('Stackable true with max_stack_size 2 passes', ...);

// Test B3.3: Nested schemas
test('Complex nested conditional', ...);
```

## Gap B4: CLI/Validator Behavior Misalignment

### Current State
**CLI Behavior:**
```javascript
// From CLI/src/validate.js:98-110
function validateSingleFile(filePath, options) {
  // 1. Detect schema from content
  // 2. Call validateFile()
  // 3. Format output
}
```

**Validator Behavior:**
```javascript
// From Validator/src/validator.js
function validateFile(filePath, schemaName) {
  // Only validates with provided schemaName
}
```

**Issue:** CLI has its own schema detection logic that may drift from validator expectations.

### Evidence
**CLI Schema Detection:**
```javascript
if (data.id.startsWith('reward_')) {
  schemaName = 'reward';
} else if (data.id.startsWith('item_')) {
  schemaName = 'item';
}
```

**Validator:** Requires explicit `schemaName` parameter.

**Gap:** If CLI detection logic changes or drifts, behavior becomes inconsistent.

### Impact
- **Severity:** Low
- **Layer:** 1+2 (Integration)
- **User Impact:** Inconsistent validation results possible
- **AJV Migration Impact:** Minor — CLI should use validator API exclusively

### Recommended Fix
```javascript
// CLI should delegate schema detection to validator
const result = validateEntity(filePath); // Let validator detect schema
```

## Newly Discovered Risks

### Risk 1: Fixture Organization
**Finding:** Current fixture organization (valid/invalid/boundary mixed) may complicate automated discovery.

**Mitigation:** Document expected structure in FIXTURE_MANIFEST.md; discovery code should handle mixed directories.

### Risk 2: Reference Validation Performance
**Finding:** `validateReferences()` builds full entity registry each call — O(n²) potential.

**Mitigation:** Cache registry in `validateProject()`; document performance characteristics.

### Risk 3: CLI Exit Code Consistency
**Finding:** Exit codes are CLI responsibility, not validator.

**Mitigation:** Add contract tests for exit codes to prevent drift.
