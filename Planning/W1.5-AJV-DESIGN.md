## 2. Layered Architecture

### 2.1 Layer 1 — JSON Schema Validation (AJV)
- **Engine:** AJV 8.x with Draft 2020-12 support
- **Responsibilities:** Type checking, required fields, enums, patterns, if/then/else, allOf/anyOf
- **Input:** Raw JSON entity
- **Output:** `{ valid: boolean, schemaErrors: [] }`
- **Error Format:** Custom wrapper to preserve current error structure (type, path, message)

### 2.2 Layer 2 — OpenMMO Semantic Validation (Custom)
- **Responsibilities:**
  - ID semantics (item_/reward_ prefix checking via reference registry)
  - Reference existence checks (B2)
  - Target validation (item→reward, reward→item/currency)
  - Circular reference detection
  - Recursive discovery (B1)
  - CLI alignment (B4)
  - Docs alignment (B5)
- **Input:** Layer 1 success result + entity data
- **Output:** `{ valid: boolean, semanticErrors: [] }`

### 2.3 Integration Flow
```
function validateEntity(filePath) {
  const schemaResult = layer1Validate(filePath);
  if (!schemaResult.valid) return schemaResult;
  
  const semanticResult = layer2Validate(schemaResult.data);
  
  return combineResults(schemaResult, semanticResult);
}
```

### 2.4 Schema/Fixture Policy
- **No new schema surface area**
- **Corrections allowed** only for truth mismatches or defects (requires QA + Spec + Chief approval)
- Fixture updates limited to defect corrections

## 3. AJV Migration Design

### 3.1 Layer 1 Implementation Steps
1. **Add AJV dependency** (`package.json`)
2. **Create validator wrapper**
```javascript
const Ajv = require('ajv');
const ajv = new Ajv({ allErrors: true, strict: false });

function layer1Validate(data, schemaName) {
  const schema = loadSchema(schemaName);
  const validate = ajv.compile(schema);
  const valid = validate(data);
  const schemaErrors = valid ? [] : formatAjvErrors(validate.errors);
  return { valid, schemaErrors, data };
}
```
3. **Error formatting** to match existing error structure:
```json
{
  "type": "REQUIRED_FIELD_MISSING",
  "path": "equipment_slot",
  "message": "Required field missing: equipment_slot"
}
```
4. **Schema loading and caching** remains unchanged
5. **Test coverage**: All existing 28 tests must pass unchanged

### 3.2 Layer 2 Implementation Steps
1. **Extract semantics** into `Validator/src/semantic.js`
2. **Functions:**
```javascript
function validateReferences(data, registry) { ... }
function detectCircularReferences(entities) { ... }
function buildEntityRegistry(files) { ... }
function layer2Validate(data, context) { ... }
```
3. **Context** includes fixture registry, discovery results
4. **Performance**: Cache registries during project validation

### 3.3 CLI Interaction
- CLI will call `validateEntity(filePath)` with schema auto-detection handled inside validator
- CLI output unchanged; contract tests ensure structure + exit codes

### 3.4 Tests/Fixtures Impact
- **Validator tests:**
  - Existing 28 tests re-run against AJV
  - Add new tests for AJV-specific behavior (pattern, enum, if/then)
- **CLI tests:**
  - Ensure CLI uses validator API only
- **Fixtures:**
  - No changes unless specific defect identified
- **Docs:**
  - Update to reflect Layer 1/Layer 2 architecture
  - Ensure examples match actual behavior (B5)

### 3.5 Migration Risks & Mitigation
| Risk | Mitigation |
|------|------------|
| AJV errors differ | Normalize error format |
| Performance regression | Benchmark pre/post |
| Schema incompatibility | Run contract tests + fixtures |
| Layer separation complexity | Document API boundaries |

## 4. Gap B3 Determination

### Investigation
- Tested equipment conditional: fails when slot missing (expected)
- Tested stackable conditional: passes even when `max_stack_size` = 1 (AJV interprets `minimum` correctly)
- No behavioral mismatch observed — only missing targeted tests

### Conclusion
- **B3 is coverage-only.** Validator behavior is correct; need expanded tests to ensure future changes don’t break conditionals.

### Action
- Add tests for:
  - Equipment slot requirement (pass/fail)
  - Stackable true with max_stack_size 1 (valid)
  - Stackable true with max_stack_size 0 (invalid)
