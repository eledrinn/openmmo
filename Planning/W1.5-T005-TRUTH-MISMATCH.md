# W1.5-T005 Truth Mismatch Report

**Date:** 2025-01-21  
**Task:** W1.5-T005 (Layer 1 AJV Implementation)  
**Discovered By:** integration-dev  
**Status:** Pending Review

---

## Mismatch Summary

| Aspect | Current State | Expected by Test |
|--------|---------------|------------------|
| Schema (item.schema.json) | `max_stack_size: { minimum: 2 }` when `stackable: true` | — |
| Test (validator.spec.js) | — | `max_stack_size: 1` should be VALID |
| AJV Behavior | Correctly rejects `max_stack_size: 1` | — |
| Old Validator | Allowed `max_stack_size: 1` (bug) | — |

---

## Evidence

**Schema Definition:**
```json
{
  "if": {
    "properties": { "stackable": { "const": true } }
  },
  "then": {
    "properties": {
      "max_stack_size": { "minimum": 2 }
    }
  }
}
```

**Failing Test:**
```javascript
test('Stackable item with max_stack_size = 1 is valid', () => {
  const item = {
    id: 'item_test',
    name: 'Test',
    item_type: 'material',
    stackable: true,
    max_stack_size: 1  // Schema says minimum: 2
  };
  const errors = validateValue(item, loadSchema('item'), '');
  assertEqual(errors.length, 0, 'Should have no errors');  // FAILS
});
```

**AJV Output:**
```
Error: Should have no errors: expected 0, got 2
```

---

## Analysis

The old custom validator had a bug in its conditional validation implementation. It was not properly enforcing the `minimum: 2` constraint when `stackable: true`.

AJV correctly enforces this constraint, revealing the truth mismatch.

---

## Options

### Option A: Fix the Test (Recommended)
Change the test to expect `max_stack_size: 1` to be **invalid** when `stackable: true`.

```javascript
test('Stackable item with max_stack_size = 1 is invalid', () => {
  const item = {
    id: 'item_test',
    name: 'Test',
    item_type: 'material',
    stackable: true,
    max_stack_size: 1  // Should be invalid
  };
  const errors = validateValue(item, loadSchema('item'), '');
  assertTrue(errors.length > 0, 'Should have errors');
});
```

### Option B: Fix the Schema
Change the schema to allow `max_stack_size: 1` when `stackable: true`.

```json
{
  "then": {
    "properties": {
      "max_stack_size": { "minimum": 1 }
    }
  }
}
```

**Not Recommended:** A stack size of 1 is the default for non-stackable items. If an item is stackable, it should have a minimum stack size greater than 1.

---

## Recommendation

**Option A: Fix the Test**

The schema is correct: a stackable item should have `max_stack_size >= 2`. The test was written expecting buggy behavior from the old validator.

This is a **test correction**, not a schema correction.
